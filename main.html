LawBot Is Here To Answer Your Queries

LawBot is an AI-based lawyer that can process textual queries and answer them accordingly with citations to the PPC (Pakistan Penal Code) and Constitution. This is especially useful for people living in remote areas and even urban women looking to get a consultation privately.

Functional Requirements

  1. User Should be able to register by providing information like email,name and password.
  2. User should be able to login by providing credentials.
  3. User should be able to logout from the system at any time.
  4. System should be able to validate data entry of a particular user.
  5. User must be able to update his profile by editing existing fields.
  6. System should have an option of choosing lawyer type like Family lawyer,Criminal Defense lawyer,Business lawyer,Estate Planning lawyer etc.
  7. System should allow users to chat i.e. user can input his query in form of a text string.
  8. System should provide feedback by informing users if a response is not clear or unavailable.
  9. System shall inform the user about spelling mistakes and will ask to rephrase his message by "autocorrect" feature.
  10. The system shall inform the user about the validity of the sentence and answer the question asked.
  11. User must be able to save the chat i.e a history/log is maintained in a database like MySQL.
  12. System should provide suggestions of questions to the user for better response.
  13. System should notify user of new court decisions that can affect his case.
  14. It should have an option of Voice-to-Text speech (for those who can't type).
  15. The chat window will show an error message if there is a network issue.
  16. Language used must be understandable preferably English will be used for now..
  17. It should be Web-Based as well as Android application.
  18. It should be integrated with a database of PPC (Pakistan Penal Code) and Constitution , to answer queries accordingly.
  19. Feedback mechanism for users to rate the conversation in order to make imrpovements.
  20. It will use API's to fetch user's information from different applications to ensure maximum engangement and flexible solutions.

Non-Functional Requirements

  1. Speed: System should give response/legal advice in fraction of seconds by fetching data from the server quickly. For slow operation, system should provide users with progress level.
  2. Confidential: All the user's information should be confidential and only available to the user currently logged in and any unauthorized access must be denied.
  3. Secure: The system should withstand malware attacks by keeping it up-to-date and patched or providing least privilege to the users.
  4. Privacy: User's data should be kept private and users have to authenticate themselves before being able to query information from the bot.
  5. Usability: The system interface should be user friendly and easy to understand a new user will make less than 3 mistakes after 5 minutes use.
  6. Robust: System should be able to recover from errors by coping with incorrect/unexpected input.
  7. Reusability: It should have chances of reusing a software component in developing a new system.
  8. Performance: Expectations regarding response time, multiple user handling capacity should be fulfilled by the system.
  9. Efficiency: The process time and resource usage of the system should be controlled and efficient.
  10. Adaptiveness: The system should have the ability to be modified, improved or scaled.
  11. Visual language must be well defined.The GUI must provide users with icons, buttons and a consistent layout.
  12. System must provide an error feedback in case of exceptions like if a user fails to complete a task.
  13. Responsiveness: System should adjust to different windows sizes like desktop/mobile phone etc. accordingly.
  14. Scalability: The system must be able to integrate with newer versions seamlessly.
  15. Availabilty: It should be available 24/7. whenever the user needs it.
  16. Portability: It should be able to run on different systems without any error.
  17. Maintainability: System should adapt to changes i.e. it shuld be ready for any fault corrections or new increments.
  18. Talking to the bot should feel like talking to a person (the bot should not reply instantly, 1-2 second delay between responses).
  19. Reliability: System should recover from errors fastly and failure rate should be less.
  20. User's expectations must be fulfilled by the system.
